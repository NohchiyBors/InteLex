# Prompt: StatusProject

Работай в режиме `StatusProject`.

Твоя задача — вести проект не только через действия, но и через устойчивое состояние проекта в файлах. В этом режиме ты обязан поддерживать актуальными `TODO`, `MEMORY`, `PROJECT-RESUME` и при необходимости `STATUS-LOG`.

## Основные правила

1. В начале каждой сессии сначала прочитай:
   - `TODO*.md`
   - `MEMORY*.md`
   - `PROJECT-RESUME*.md`
   - при наличии `STATUS-LOG*.md`
2. Не начинай работу вслепую, если уже есть сохраненный контекст.
3. После анализа явно определи:
   - текущую цель
   - следующий практический шаг
   - что заблокировано
4. После каждого заметного действия обновляй проектные файлы, а не оставляй состояние только в переписке.
5. Если что-то завершено, помечай это в `TODO` чекбоксом `[x]`.
6. Если появляется новое ограничение, риск, правило, ссылка, зависимость или договоренность, запиши это в `MEMORY`.
7. Если сессия заканчивается или найден важный промежуточный итог, обнови `PROJECT-RESUME`.
8. Если задача длинная, многошаговая или идет пакетами, фиксируй ход выполнения в `STATUS-LOG`.

## Как вести файлы

### TODO

Используй `TODO` как живой оперативный список:
- что уже завершено
- что в работе сейчас
- что идет следующим шагом
- какой текущий блокер
- какие файлы и артефакты критичны

### MEMORY

Используй `MEMORY` только для устойчивого контекста:
- бизнес-правила
- технические ограничения
- формулы
- принятые решения
- пути к важным данным
- внешние зависимости
- что нельзя ломать

Не складывай туда одноразовый шум.

### PROJECT-RESUME

Используй `PROJECT-RESUME` как короткую выжимку для старта следующей сессии:
- дата
- что за проект
- где остановились
- какой следующий шаг
- какие результаты уже подтверждены
- что мешает продолжению

### STATUS-LOG

Используй `STATUS-LOG`, если:
- есть серия одинаковых операций
- есть ретраи
- есть пакетная обработка
- есть импорт/экспорт/миграция
- нужно видеть историю выполнения по времени

## Приоритет порядка обновления

1. `TODO` — всегда
2. `MEMORY` — когда появился устойчивый факт
3. `PROJECT-RESUME` — при существенном прогрессе и в конце сессии
4. `STATUS-LOG` — при длинном процессе или серии операций

## Формат поведения

- Пиши кратко и предметно.
- Не дублируй одно и то же во все файлы.
- Не оставляй завершенные этапы только в сообщении пользователю.
- Если контекст изменился, сначала обнови файлы, потом резюмируй.
- Если есть конфликт между старым состоянием и реальной ситуацией, исправь файлы под текущее состояние.

## Правило завершения сессии

Перед завершением сессии проверь:
- `TODO` отражает реальный следующий шаг
- `MEMORY` содержит новые постоянные правила
- `PROJECT-RESUME` позволяет продолжить работу без перечитывания всей истории
- `STATUS-LOG` обновлен, если процесс был поэтапным
