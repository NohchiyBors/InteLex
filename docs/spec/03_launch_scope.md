# InterLex: Этап 1 запуска и этап 2 развития

## 1. Цель этапа 1 запуска

Этап 1 запуска должен вывести в продакшн полноценный корпоративный сайт, который:
- объясняет позиционирование InterLex
- даёт полноценный каталог услуг как основную точку входа в коммерческие разделы
- продаёт ключевые услуги
- собирает лиды
- индексируется поисковыми системами
- поддерживает поэтапную мультиязычность с базовым запуском на `RU` и `EN`

Этап 1 запуска не должен перегружаться второстепенными функциями. Приоритетом остаются полная коммерческая структура сайта, рабочая лидогенерация, SEO-скелет и управляемый контент.

## 6. Рекомендуемый стек этапа 1

- `Next.js + TypeScript + Tailwind`
- Headless CMS
- Простая форма с отправкой в email/CRM
- Базовая аналитика
- Контентная модель под масштабирование

Если нужен быстрый запуск без сильной кастомной логики, можно использовать и `WordPress`, но только если команда сознательно жертвует гибкостью и частью производительности.

## 7. Критерий готовности этапа 1

Этап 1 готов к запуску, если:
- все ключевые страницы опубликованы
- формы работают
- есть полноценная `RU`-версия для `interlex.kz`
- есть полноценная `EN`-версия для `interlex.ge`
- структура локализации готова под `ZH`, `KK`, `KA`
- SEO-метаданные заполнены
- аналитика подключена
- сайт не ломается на мобильных
- структура контента позволяет дальше масштабировать SEO
