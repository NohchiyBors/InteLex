# StatusProject: InterLex Website

Этот файл используется как входная точка в режим `StatusProject` для проекта `InterLex`.

## Актуальные статусные файлы
- [TODO.md](D:\Data\OneDrive\source\www\InteLex\TODO.md) — текущие задачи, рамки, ближайшие действия
- [MEMORY.md](D:\Data\OneDrive\source\www\InteLex\MEMORY.md) — устойчивые решения, правила, зависимости и факты
- [PROJECT-RESUME.md](D:\Data\OneDrive\source\www\InteLex\PROJECT-RESUME.md) — краткий снимок состояния проекта на текущую сессию

Журнал по шагам при длинных задачах ведите **локально** (не в git): скопируйте `StatusProject/templates/STATUS-LOG.template.md` в корень как `STATUS-LOG.md` — файл перечислен в `.gitignore`.

## Порядок чтения в начале сессии
1. Открыть `TODO.md`
2. Открыть `MEMORY.md`
3. Открыть `PROJECT-RESUME.md`

## Правила ведения
- После каждого заметного шага обновлять статусные файлы, а не оставлять состояние только в чате.
- Не использовать устаревшую рамку `MVP` как основную терминологию проекта.
- Сохранять согласованную проектную модель:
  - `InterLex` — основной бренд
  - `interlex.kz` — `RU-first`
  - `interlex.ge` — `EN-first`
  - `ZH / KK / KA` — дополнительные локали
  - сайт — полноценный corporate website с каталогом услуг, не landing page
